# Contributing

## Code of Conduct

This project follows the Contributor Covenant Code of Conduct.
All participants are expected to uphold this code.

## Getting Started

1. Fork the repository.
2. Clone your fork locally.
3. Create a feature branch from `main`.
4. Make your changes and add tests.
5. Run the test suite with `npm test`.
6. Submit a pull request.

## Coding Standards

- Use TypeScript strict mode.
- Format code with Prettier (default config).
- Lint with ESLint before committing.

## Commit Messages

Follow the Conventional Commits specification:

```
<type>(<scope>): <description>
```

Types: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`.

## Pull Request Process

1. Ensure all CI checks pass.
2. Request review from at least one maintainer.
3. Address all review comments before merging.
4. Squash commits into a single logical change.
